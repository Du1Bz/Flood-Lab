# Flood-Lab プロジェクト概要

## なぜ作るか

Flood（Notion同期ツール）を運用する中で見えてきた限界を解決するために作る。  
Flood-Lab は Flood の「反省から生まれた後継可視化ツール」。

---

## Flood の反省点

### 1. Notion はグラフが弱い

散布図が使えない。折れ線グラフも自由度が低く、複数指標の重ね合わせができない。  
KDA vs Accuracy の相関を見たい、eMMR の推移に移動平均を重ねたい、といった分析ができなかった。

→ **Flood-Lab は Plotly を使い、インタラクティブなグラフを自由に作る。**

### 2. 欲しいデータを後から追加できなかった

OpenSpartan の JSON から「その時点で欲しい値」だけ抜いて Notion に書き込んでいた。  
後から「あの指標も見たかった」となっても、過去の試合データを再取得・再加工しなければならなかった。

→ **Flood-Lab は JSON を捨てずに全フィールドを DataFrame に展開する。**  
→ **「取捨選択はビューで行う。ストレージには全部入れる。」**

### 3. フィルタでデータを捨てていた

途中抜け試合・1分未満の試合などを取り込み段階で除外していた。  
統計上ノイズになるデータでも、「記録として存在した事実」は残すべきだった。

→ **Flood-Lab は全試合を取り込む。**  
→ **フィルタは「統計・集計から除外する」だけ。データそのものは捨てない。**  
→ **除外対象には `exclude_flag`（除外フラグ）を立て、ビュー側で絞り込む。**

---

## 設計思想

### データは捨てると戻らない

取り込み段階での判断は最小限にする。  
「これは要らない」という判断は、分析してみないとわからない。

### フィルタは除外フラグで表現する

| 除外理由 | フラグ値 |
|---|---|
| 試合時間が1分未満 | `short_match` |
| 途中抜け（完走していない） | `incomplete` |
| 手動除外 | `manual` |

集計・インサイトは `exclude_flag` が空の試合のみを対象にする。  
ただし除外フラグが立った試合も試合履歴ビューには表示する。

### ビューで絞る、ストレージでは絞らない

```
❌ Flood の方法：取り込み時にフィールドを選んで捨てる
✅ Flood-Lab の方法：全フィールドを展開し、表示時に絞る
```

## 今後の拡張候補

### XUID → ゲーマータグ変換

Halo の公開 API で XUID からゲーマータグを引ける。  
現状は XUID の数値で表示しているプレイヤー名を、ゲーマータグに変換して表示したい。

- 実装先: `src/utils/gamertag.py`（新規）
- キャッシュ: 一度引いたXUID→ゲーマータグのマッピングをローカルに保存
- Phase 3 以降で実装予定

### 試合履歴ビュー

1試合ずつの詳細を時系列で確認できるビュー。  
マッチ一覧テーブルとは別に、1試合クリックで詳細展開できる形が理想。

- 除外フラグが立った試合も含めて全件表示
- フィルター連動（区分 / マップ / ルール / 期間）
- Phase 2 以降で実装予定

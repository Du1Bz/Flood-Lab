## 2. レート系指標

| 指標 | 実装区分 | 計算式 | 備考 |
|---|---|---|---|
| K/D | 再実装 | `キル / デス`（デス=0のとき `キル`） | キルレシオ |
| KDA | 再実装 | `キル - デス + (アシスト / 3)` | アシスト3回をキル1回相当とみなす |
| 命中率 | 再実装 | `命中数 / 発射数` | 0除算は0 |
| パーフェクト率 | 再実装 | `パーフェクトキル / キル` | キル=0のとき0 |
| キル効率 | 再実装 | `与ダメージ / (キル + アシスト/3)` | 1キル相当あたりの与ダメージ。低いほどクリーンキル |
| デス効率 | 再実装 | `被ダメージ / デス` | 1デスあたりの被ダメージ。高いほど粘れている |

## 4. TrueSkill2 系指標

TrueSkill2 は Microsoft が Halo に採用しているスキル評価システム。  
各プレイヤーに「期待キル数」「期待デス数」を算出し、実績との乖離でスキルを更新する。

| 指標 | 実装区分 | ソース / 計算式 | 説明 |
|---|---|---|---|
| 期待キル | 移植 | `PlayerMatchStats.MMRs[].ExpectedKills` | TrueSkill2 が試合前に予測したキル数 |
| 期待デス | 移植 | `PlayerMatchStats.MMRs[].ExpectedDeaths` | TrueSkill2 が試合前に予測したデス数 |
| K-RPI | 再実装 | `キル / 期待キル` | 期待を超えたキル達成率。1.0 = 期待通り、>1.0 = 期待超え |
| D-RPI | 再実装 | `期待デス / デス` | 期待より少ないデス率。>1.0 = 期待より死ななかった |
| LGAI | 再実装 | `敵チームMMR - 試合前CSR` | ロビー格差補正インパクト。正 = 格上相手 |
| インパクトスコア | 再実装 | `(K-RPI + D-RPI) / 2` | TrueSkill2 基準の総合パフォーマンス指標 |
| 生存貢献度 | 再実装 | `(期待デス - デス) × (敵チームMMR / 試合前CSR)` | 格上相手に死なずに立ち向かった度合い |

### K-RPI / D-RPI の解釈ガイド

```
K-RPI = 1.0  → 期待通りにキルできている
K-RPI < 1.0  → 期待より取れていない（アグレッション不足 or ポジション問題）
K-RPI > 1.0  → 期待を上回るキル（ポップオフ）

D-RPI = 1.0  → 期待通りの死に数
D-RPI > 1.0  → 期待より死ななかった（生存力が高い）
D-RPI < 1.0  → 期待より多く死んでいる（立ち回りの問題）
```

## 5. eMMR 系指標

Flood / Flood-Lab 独自の推定MMR（TrueSkill2 の公開値を元に独自計算）。

| 指標 | 実装区分 | 説明 |
|---|---|---|
| eMMR v1（試合前） | 移植 | 区分別の試合前推定MMR（単純加重平均） |
| eMMR v1（試合後） | 移植 | 区分別の試合後推定MMR |
| eMMR v2 | 移植 | カルマンフィルタ適用後の平滑化eMMR。ノイズを除いたトレンドを表す |
| eMMR v2 sigma | 移植 | eMMR v2 の不確実性（カルマン分散）。試合数が少ないほど大きい |
| AvgCSR20 | 移植 | 直近20試合の平均CSR。eMMR v1 算出に使用 |

### eMMR v2（カルマンフィルタ）のパラメータ

```python
# Flood の metrics.py より移植
PROCESS_NOISE = 100      # Q: プロセスノイズ（MMRの真の変動の大きさ）
MEASUREMENT_NOISE = 500  # R: 測定ノイズ（1試合のMMR変動のばらつき）
INITIAL_SIGMA = 1000     # 初期不確実性
```

## 7. パーティ系指標

| 指標 | 実装区分 | 計算式 | 説明 |
|---|---|---|---|
| パーティ人数 | 移植 | `build_party_map()` による推定 | 1=ソロ, 2=デュオ, 3=トリオ, 4=フルパ |
| パーティタイプ | 再実装 | パーティ人数をラベル化 | "ソロ" / "デュオ" / "トリオ" / "フルパ" |
| ソロフラグ | 再実装 | `パーティ人数 == 1` | bool |
| パーティフラグ | 再実装 | `パーティ人数 > 1` | bool |
| 勝敗フラグ | 再実装 | `勝敗 == "勝ち"` → 1, else 0 | 集計用 |

### パーティ検出ロジック

- カスタムゲーム: 4人固定
- ランク試合: 前後2試合を含むウィンドウ（最大5試合）で、同チームに **3回以上** 登場した味方 = パーティメンバー
- 詳細は `src/logic/parser.py:build_party_map()` を参照

## 8. 時間正規化指標

試合時間が異なるモード（Slayer vs Strongholds等）を横断比較するための指標。

| 指標 | 実装区分 | 計算式 | 備考 |
|---|---|---|---|
| 試合時間(分) | 再実装 | `秒 / 60` | 計算基底。小数点あり（例: 7:30 → 7.5） |
| KPM | 再実装 | `キル / 試合時間(分)` | Kills Per Minute |
| DPM | 再実装 | `デス / 試合時間(分)` | Deaths Per Minute |
| 与ダメージ/分 | 再実装 | `与ダメージ / 試合時間(分)` | |
| 被ダメージ/分 | 再実装 | `被ダメージ / 試合時間(分)` | |
| エンゲージメント密度 | 新規 | `(キル + デス + アシスト) / 試合時間(分)` | 試合への関与の濃さ。ルール間比較に有用。Slayerは高め、Oddball/Strongholdsは低くなる傾向 |

## 9. パワーウェポンコントロール系指標

Halo Infiniteの競技シーンにおいて、パワーウェポン（PW）の確保はマップコントロールと直結する。
自チームと敵チームのPWキル数を比較することで、疑似的なマップコントロール率として使える。

| 指標 | 実装区分 | 計算式 | 備考 |
|---|---|---|---|
| 自チームPWキル | 新規 | `MatchStats.Teams[自チーム].Stats.CoreStats.PowerWeaponKills` | 生値 |
| 敵チームPWキル | 新規 | `MatchStats.Teams[敵チーム].Stats.CoreStats.PowerWeaponKills` | 生値 |
| PWコントロール率 | 新規 | `自チームPWキル / (自チームPWキル + 敵チームPWキル)` | 両チームとも0のときNone。0.5が均衡、>0.5が優勢 |

### PWコントロール率の解釈ガイド

```
> 0.6  → 明確なPW優勢。マップコントロールを握れている
= 0.5  → 均衡。PWの奪い合いが五分
< 0.4  → PW劣勢。相手にPWを活かされている試合
None   → 両チームPWキル0（PWが存在しないマップ等）
```

## 10. 集計指標

`processor.py` が `groupby` で生成する集計値。Notion のロールアップ相当。

### 集計軸

単軸・多軸の組み合わせに対応する。

| 集計軸 | Notion 相当 |
|---|---|
| セッション単位 | セッションDB のロールアップ |
| 週単位 | 週次サマリー |
| 月単位 | 月次サマリー |
| 区分別 | 区分DB のロールアップ |
| マップ別 | マップDB のロールアップ |
| ルール別 | ルールDB のロールアップ |
| **区分 × マップ × ルール** | アナリシスDB のロールアップ |

### 集計する指標

| 集計指標 | 計算方法 |
|---|---|
| 試合数 | `count()` |
| 勝利数 | `sum(勝敗フラグ)` |
| 勝率 | `mean(勝敗フラグ)` |
| 平均K/D | `sum(キル) / sum(デス)` ※ 平均の平均ではなく合計ベース |
| 平均KDA | `mean(KDA)` |
| 平均命中率 | `sum(命中数) / sum(発射数)` ※ 合計ベース |
| CSR増減合計 | `sum(CSR増減)` |
| 平均与ダメージ | `mean(与ダメージ)` |
| 平均被ダメージ | `mean(被ダメージ)` |
| 平均ダメージ差 | `mean(ダメージ差)` |
| 平均チーム内順位 | `mean(チーム内順位)` |
| ソロ勝率 | `sum(ソロ勝利フラグ) / sum(ソロフラグ)` |
| パーティ勝率 | `sum(パーティ勝利フラグ) / sum(パーティフラグ)` |
| K-RPI（集計） | `sum(キル) / sum(期待キル)` ※ 合計ベース |
| D-RPI（集計） | `sum(期待デス) / sum(デス)` ※ 合計ベース |

> **注意**: K/D・命中率・K-RPI などは「合計ベース」で計算する。  
> `mean(K/D)` は試合時間の長短に引きずられる。
